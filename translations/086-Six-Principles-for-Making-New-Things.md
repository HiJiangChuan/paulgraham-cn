
# 创造新事物的六条原则  
2008年2月  

[Arc](https://hijiangchuan.com/paulgraham/EXTRA009-Arc) 发布后激起的强烈反应，带来了一个意想不到的结果：它让我意识到，我其实有一套设计哲学。那些表达清晰的批评者的主要抱怨是：Arc 看起来太轻飘飘了。多年努力之后，我所能展示的成果竟然只是几千行宏代码？他们质疑我为什么不去解决更“实质性”的问题。  

当我反复思考这些评论时，忽然觉得这种论调很熟悉。这正是人们当初对 Viaweb、对 Y Combinator、甚至对我大多数文章的评价。  

当我们创立 Viaweb 时，风险投资人和所谓的电商“专家”都觉得那简直是个笑话。1995 年时，“在公寓里干活的两个人”并不像现在这样显得酷。而我们做出来的东西，在他们眼里，甚至算不上软件。在他们的定义中，“软件”意味着庞大臃肿的 Windows 应用程序。而 Viaweb 是他们见过的第一个“网页应用”，他们觉得那不过就是个网站。当他们发现 Viaweb 连信用卡支付都不处理时（我们上线的第一年确实没有处理交易），他们更是嗤之以鼻。对他们来说，交易处理才是电商的核心——那才“严肃”“复杂”。  

然而，出人意料的是，Viaweb 最终碾压了所有竞争者。  

[YC（Y Combinator）](http://ycombinator.com) 早期也几乎遭遇了同样的反应——轻得可笑。在他们看来，“创业投资”意味着 A 轮融资：数百万美元投给少数有资历的人创立的公司，经过数月严肃的商务会议，并签订厚达30厘米的合同。而 YC 看起来毫无分量。现在说 YC 是否会像 Viaweb 一样成功还为时尚早，但从众多模仿者的出现来看，显然不少人认为我们抓住了某种东西。  

我没法用除了访问量以外的方式来衡量我的文章是否成功，但至少公众的反应已经变了。最初，Slashdot 上的喷子们的默认反应大致可以翻译为：“这家伙是谁？他凭什么写这些？我虽然没读，但这么短、这么随意的文章，绝不可能在这些有学位专家都写了厚书的话题上讲出什么有价值的东西。”如今，一代喷子已去，新一代在新网站上继续活跃，但他们至少省略了开头那句“这家伙是谁”。  

现在，人们对 Arc 的反应，与他们当初对 Viaweb、YC、以及我文章的反应如出一辙。为什么会这样？我意识到答案其实在于：这四者的“作风”（m.o.）完全一样。  

我喜欢寻找：
一、**简单的解决方案**
二、**用于解决被忽略的问题**
并且
三、**这些问题确实需要解决**
四、**以尽可能非正式的方式发布**
五、**从极其粗糙的版本1开始**

最后
六、**快速迭代**  

当我第一次明确地写下这些原则时，我注意到一个惊人的事实：这几乎就是一套**必然引发蔑视反应的配方**。简单方案虽然更好，但看起来不够“高级”；被忽略的问题本身就是别人觉得“不重要”的问题；以非正式方式发布意味着人们不能靠表面判断，必须真正理解你做了什么——这需要动脑子；而一个粗糙的版本1，总是显得小而不完整。  

我当然早就注意到：人们从来不会立刻理解新想法。我原以为那只是因为他们笨。现在我发现，事情没那么简单。这种做法简直就是专为击中“日常愚蠢”而设计的。就像逆向投资基金，这种方法几乎总让你做出“在别人看来是错的”事。  

但正如逆向投资一样——**这正是关键所在**。这种策略之所以长期有效，是因为它让你获得别人放弃的所有优势。  
- 如果你解决被忽略的问题，你就更容易发现新东西，因为竞争更少。  
- 如果你用非正式的方式发布，你就（a）省去了让它看起来“体面”的巨大成本，（b）避免了连自己都被外表欺骗的风险。  
- 如果你发布粗糙的版本1并不断迭代，你的方案就能受益于“自然界的想象力”——正如费曼所说，那远比人类的想象力强大。  

在 Viaweb 的例子里，简单方案是让软件跑在服务器上；被忽略的问题是“自动生成网站”——1995 年时，所有在线商店都是人工设计的，而我们知道那根本无法扩展；真正重要的环节其实是**网页设计**，而非交易处理。非正式的发布方式就是我——穿着牛仔裤和T恤直接上门推销。而那个粗糙的版本1，据我记得，代码量不到一万行。  

这种方法的威力不仅适用于创业、编程语言或写作，它几乎适用于任何创造性工作。绘画也可以，比如塞尚（Cézanne）和克利（Klee）正是这么做的。  

在 Y Combinator，我们甚至用真金白银下注，鼓励创业者按这套方式做事。新想法往往就在你鼻子底下。  
寻找那些**别人忽略的简单事物**——那些后来会被人称为“显而易见”的东西——尤其是当人们被过时的习惯或表面的“高大上”误导时。搞清楚**真正的问题是什么**，并确保你解决的是它。别在意企业化的外表，**长期胜出的永远是产品本身**。尽早发布，从用户那儿学习你真正该做什么。  

[Reddit](http://reddit.com) 就是这种方法的典范。当 Reddit 刚上线时，看起来几乎什么都没有。对于视觉不敏感的人来说，它那极简的设计简直像是根本没设计。但 Reddit 解决了真正的问题：告诉用户“什么是新的”，然后尽量不妨碍他们。结果它变得极度成功。如今当常识追上它，人们回头看会觉得“显而易见”。他们以为 Reddit 的创始人运气好。可像所有这类事一样，**那其实比看上去难得多**。Reddit 的创始人当年奋力逆流而上，最终改变了水流方向——现在看起来，他们不过是顺流而下。  

所以，当你看到 Reddit 这样成功的产品，心想“真希望我也能想到这样的点子”时，请记住：这样的想法其实**到处都是**。只是你忽略了它们——因为它们**看起来不对**。
